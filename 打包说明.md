## 🛠️ 配置修改

### 修改签名配置

编辑 `signature_config.json`：

```json
{
  "signature": {
    "enabled": true,
    "default_certificate": "sap_hour_operation",
    "timestamp_server": "http://timestamp.digicert.com",
    "certificates": {
      "sap_hour_operation": {
        "name": "SAP工时操作工具证书",
        "sha1": "144ac4069565211ab67d25a9d6d33af0e18e511e",
        ...
      }
    }
  }
}
```

### 修改打包配置

编辑 `build_with_signing.py` 中的 CONFIG 字典：

```python
CONFIG = {
    'main_script': 'Sap_Hour_Operate_theme.py',
    'signature_config': 'signature_config.json',
    'certificate_file': '170859-code-signing.cer',
    'icon_file': 'ch-4.ico',
    'exe_name': 'Sap_Hour_Operate_theme',
    'project_certificate': 'sap_hour_operation',
    'console': False,  # 是否显示控制台
}
```

## ❓ 常见问题

### Q1: code_signer 模块导入失败？
**A:** 确保以下文件存在：
```
code_signer/__init__.py
code_signer/core.py
code_signer/config.py
signing_tool.py
signature_config.json
```

### Q2: 签名失败，提示证书不存在？
**A:** 检查以下几点：
1. 证书文件 `170859-code-signing.cer` 是否存在
2. 证书是否已安装到 Windows 证书存储
3. signature_config.json 中的 SHA1 指纹是否正确

### Q3: 打包后运行提示缺少模块？
**A:**
- 在 spec 文件的 `hiddenimports` 中添加缺失的模块
- 或在打包命令中添加 `--hidden-import=模块名`

### Q4: 打包文件太大？
**A:**
- 使用 UPX 压缩（已默认启用）
- 在 spec 文件的 `excludes` 中排除不需要的模块
- 考虑使用虚拟环境最小化依赖

### Q5: Windows Defender 误报？
**A:**
- 使用正式的代码签名证书签名 ✅
- 添加微软应用程序信誉例外
- 提交文件到微软 SmartScreen 白名单

### Q6: 如何手动签名？
**A:** 使用 signtool：
```bash
signtool sign /f 170859-code-signing.cer /fd SHA256 ^
    /tr http://timestamp.digicert.com /td SHA256 ^
    dist\Sap_Hour_Operate_theme.exe
```
